untiy -----LOD (level of detail)

Level of detail —— 多层次细节

性能优化时，我们有时需要对物体进行处理，来让游戏跑起来更流畅

多层次细节处理，就是让一个物体，在相机距离不同的情况下，显示不同的模型，从而节省性能的开销

开发者做个非常精致的房子，顶点数/面数一定很多

玩家在游戏中，控制人物相机视角离一座房子近的时候，看起来非常精致好看！游戏体验很好！(假设性能开销为100)

当玩家走远的时候，这座房子还在那里，但已经走了很远了，房子看起来不是那么清晰了

但由于房子的模型精度太高无论远近，模型的顶点和面数都不变

Gpu都需要去处理，并绘制房子的图形效果。性能开销还是100

那么就有些浪费性能了，当玩家走远的时候，我们只需要让玩家看到一个房子还在那里即可，因为无论精细度多高，玩家看起来房子都很小，看不清

这是一个房子还好说，现如今没有什么cpu绘制不了这么一个房子。但游戏对象肯定不止一个，大型游戏可能有一堆，几百个，上千个物体

所以作为开发者，我们无需把一个精度那么高的模型，还放在那里，给CPU GPU增加工作量

这时候我们就可以用 LOD 技术，来对模型进行 分层级显示

视角离近时，我们让物体显示精细度高的模型

视角离远时，我们让物体显示精细度低的模型

这就是 Level of detail 多层次细节优化

对于unity中脚本参数的理解

1、OnTriggerEnter函数中有一个叫做col的参数，它保存了碰撞事件的所有信息。而在函数内部，我们首先检测到与触发器碰撞的游戏对象是不是玩家（“Player”）,这里我i们检测的是标签。（注意：Player 标签是unity 自带的，无需我们手动添加，但是需要我们把标签“贴到”我们的玩家对象上。）

现在，我们来理一下我们代码的逻辑；

每当玩家拾取能源PowerCell（也就是与之触发器碰撞器发生碰撞）时，PowerCell对象会调用自身脚本中的OnTriggerEnter()函数，这个函数首先检测与之发生碰撞的对象是否是玩家，如果是，就发生一个消息给玩家对象的脚本Inventory，调用它的一个名为CellPickup()的函数。这个函数会做三件事情：

●首先，播放一个拾取能源的音频剪辑；

●接着，更改charge变量，也就是更改当前收集到的能源数，每拾取一个能源，能源数增加1；

●最后，根据玩家收集到的能源数（charge变量的值）来为PowerGUI对象选择对应纹理。

unity中的粒子系统

untiy的粒子系统之所以被称为粒子系统，而不被称为组件（我们前面说过，unity都是以组件的形式实现其功能，）是因为它需要三个组件一起工作，才能使粒子系统正常运作。

  • 粒子发射器： 这个组件决定粒子的生成、存活时间、速率和范围。

  • 粒子动画器： 这个组件声明粒子在存活时间内的行为。

  • 粒子渲染器： 这个组件声明粒子的渲染外观。

    简而言之，粒子发射器创造粒子，粒子动画器声明粒子的行为和多样性，粒子渲染器通过使用材质和各种参数决定粒子的外观。

unity 粒子系统的主要步骤

1、创建空物体

2、为空物体添加粒子系统组件

3、设置组建参数

4、创建粒子系统的材质，并加到空物体上（则此材质会自动应用到每个粒子片上）

5、为空物体增加音频组件，并设置火焰声音

6、将此空物体放置到场景中合适的地方

  • 注意： 从上面的步骤来看，粒子效果的展示是需要依托一个空物体而展开的。而粒子系统效果是以组件的形式来添加的。

一个至关重要的函数是OnGUI()，几乎所有与界面相关的工作都是在这个消息处理函数中进行的。

装箱和拆箱

装箱

装箱就是值类型到 object 类型或者到该值类型所实现的接口类型所实现的一个隐式转换过程（可显式）。装箱的时候会在堆中自动创建一个对象实例，然后将该值复制到新对象内。

　　从图可知，对象 o 存的是地址引用，指向的是堆上的值，这个值的类型和变量 i 一样，也是 int 类型，值（123）也就是从变量 i Copy 过来的一个副本值而已。

　　【备注】装箱默认是隐式的，当然，你可以选择显式，但这并不是必须的。

拆箱

　拆箱是从 object 类型到值类型，或从接口类型到实现该接口的值类型的显式转换的一个过程。

　　拆箱：检查对象实例，确保它是给定值类型的一个装箱值后，再将该值从实例复制到值类型变量中。

要在运行时成功拆箱值类型，被拆箱的项必须是对一个对象的引用，该对象是先前通过装箱该值类型的实例创建的。

拆箱时需要注意，转换出现异常的情形：

　　虽然，decimal 类型可以直接强转为 int 类型，但从调式的结果来看，拆箱时是会引发“转换无效”的异常。要记住，拆箱时强转的值类型，应以装箱时的值类型一致。

装箱就是把值类型转换成引用类型使用，而拆箱就是把引用类型转换成值类型使用。

C++或者c中的gets函数和puts函数

puts函数

puts()函数用来向标准输出设备（屏幕）输出字符串并换行，具体为： 把字符串输出到标准输出设备，将‘\0’转换为回车并换行。其调用方式为，puts(s): 其中s为字符串字符（字符串数组名或字符串指针）

gets()函数

gets 从标准输入设备读字符串函数，其可以无限读取，不会判断上限，以回车结束读取，所以程序员应该确保buffer的空间足够大，以便在执行读操作的时候不会发生 溢出的情况。

异步调用

不论是桌面客户端还是Web应用通常会出现处理时间较长的操作，为了在这段时间内不影响用户与应用之间的交互体验，开发人员通常会使用异步调用技术，使得比较复杂的逻辑操作由异步进行，用户仍然可以继续使用应用，不会有无响应的等待情况出现。

异步开发，就是利用线程技术,当执行一个占用时间长的任务时,不会阻止用户其它工作,而且其完成时会通知用户

QT之事件

对于qt事件接受与否的理解，比如说点击串口的X号，弹出一个messagebox提示窗口。

可以提示你，要不要关闭窗口，也就是说鼠标的点击是一个事件，e由event() 函数，传递事件，然后参数中的E->accept() 和 E->ignore()函数，，，一个是接受事件的信号，之后不再往下传了，另一个就是忽略信号了。

event 函数

event（）函数的作用就是事件分发了， 比如说 QWIdget ->Event（）—》传递 给鼠标事件或者说传递给 键盘事件，

参数e 有一个type（），通过type（）返回的值，来判断到底e触发的是哪一种事件，是QEvent::close ，那么就会执行closeevent(e).break 。

如果是QEvent::MouseMove， 那么就会执行mouseMoveEvent(e).

事件过滤器

事件过滤器主要是，在那个一个一个event事件之前, 定义的一个过滤器，，可以判断对哪个事件进行处理，对哪个事件不进行处理。

事件过滤器和被安装过滤器的组件必须在 同一个线程，否则。过滤器将不起作用。另外，如果在安装过滤器之后，这两个组件到了不同的线程，那么，只有等到二者重新回到同一线程的时候过滤器才会有效。

小总结

QT 的事件是整个QT框架的核心机制之一，也比较复杂。说它复杂，更多是因为它涉及到的函数众多，而处理方法也很多。

1、文件预处理模块

以ascii码的形式读入STEP中性交换文件，将整个文件数据读入到一个字符串数组中。在step中性文件的数据段中每一行字符串表示的是一个几何形状特征。通过循环分别提取出数据段每个几个何形状特征的相关信息。对于每个实体特征数据的提取采用字符串分割处理的方法。整个数据段提取信息流程：：

经过这个步骤，将信息分割成各个部分，简化文件的信息后将各个分割段的信息分类保存。

将提取的信息以结构体的形式保存。将分割后的字符串输入到对应的结构体中对应成员变量中，完成信息的预处理。

2、实体对象的构造和存储

预处理的功能就是将文件中表述的各个实体以结构体的形式保存在内存中，由于目前 在文件中各个实体是独立的，因此需要将这些实体按照STEP的数据模型重新组合。

也就是说建立起，根据预处理阶段的结构体中的实体关键字，查找关键字在内存中所对应的c++类的实体描述，建立与c++的描述的映射关系。将一些参数传递，比如实体名称、实体属性值等，作为参数传递给对应类的参数构造函数，构造c++实体对象。

step实体在内存中的数据结构

STep实体在内存中通过c++中的链表结构来实现其数据结构。点类、矢量类、线类的实例化后，就将每类的实体都保存在一个表内保存，实体之间存在一定的拓扑关系，在内存中通过链表的连接路径来描述实体间的拓扑关系。

最后所有的实体实例化之后形成一个关于实体的链表形式的数据结构。

特征提取

特征提取主要是将保存在内存中的STEP各个实体信息经过特征识别之后，提取出能描述该特征的数据信息。特征提取可以分为两个部门：特征识别、特征提取。

根据B_REP表示法原理，从“CLOSED_STHELL”入手识别。在STEP中性文件中一个CLOSED_STHELL实体是由多个advanced_face面拼合而成的，而advanced_face又是由曲面的参数与边界类型构成的，因此可以通过识别advancde_face的类型和数量来识别出表达的实体。

通过分析，可以看出STEP文件中的圆柱体是由两个SuRFACE曲面和两个PLANE平面，共计四个Advanced_face组成的，我们将这些特征作为识别圆柱体的规则。

识别过程如下：遍历STEP中间数据文件中的Closed_shell实体类，判断其中advanced_face的数量，再通过判断advanced_face中曲面的关键字，最终判别描述该实体是否为 圆柱体。

特征识别子模块的工作对象是前步已经实例化的各个实体对象。要按照程序的链表式识别简单实体特征，并得到该实体的特征值。

比如说一个球体：是由两个形状一样的半球面构成的，访问该球面的实例对象PSPHERICAL_SURFACE读取定形数据和定位数据。

得到完整的球体的造型数据模型并进行串行化处理。

数据转换（自定义函数）

在特征识别模块所得到的简单实体的造型数据式通过分析提取出来并实例化 给特定的类的，它可以另外存储并产生一个中间文件，然后根据不同的应用系统再设计各自的数据转换器。

特征提取的过程：

（1）从球面实体实例提取特征值：定位数据和半径

（2）建立球体的显示颜色和材质等辅助属性

（3）创建图形对象并调用绘图函数。

小结：

文件预处理模块完成STEP中性文件的信息采集；实体对象的构造模块将中性文件所表达的实体信息输入到C++表示的对应实体中，完成C++实体类的实例化；特征提取模块根据一定的算法完成基本实体的识别，并提取实体的属性值。数据转换模块将已经实例化的实体信息经过内部函数调用等操作转化成之后要使用的数据格式。
